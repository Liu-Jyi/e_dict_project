什么是项目
  一般指软件项目:一个具有完整的功能的代码集合.
  接口(API Application Program Interface):
    函数,类,模块都可能是接口.提供的具有一定功能的代码,通过获取数据后返回相应的接口.

软件项目开发流程
  需求分析 --> 概要设计 --> 项目规划 --> 详细设计 --> 
  编码测试 --> 项目测试 --> 项目发布 --> 后期维护

  需求分析:确定用户真实需求,做项目评估
    1.确定用户真实需求,项目的基本功能
    2.对项目整体进行评估(可行性,难度等)
    3.完成需求分析文档,用户确认

  概要设计:对项目进行整体设计,技术分析
    1.确定功能结构
    2.进行技术分析,构建整体框架图
    3.确定技术思路和使用技术
    4.完成概要设计文档,直到开发流程

  项目计划:确定开发的时间轴和流程
    1.确定开发的先后顺序
    2.确定时间轴,里程碑事件
    3.人员的分工
    4.形成甘特图和思维导图,设定计划

  详细设计:项目的具体实现手册
    1.详细记录项目的思路,逻辑,功能,技术点,
      代码说明,数据结构等
  
  编码测试:按照计划编写功能代码,做基本测试
    1.写代码,测代码
    2.技术攻关

  项目测试:对项目进行整体测试,调整修改
    1.做项目的各种功能和逻辑测试
    2.完成测试报告
    3.进行项目修改

  项目发布
    1.交付用户进行使用
    2.编写说明文档

  后期维护
    1.bug的修改
    2.产品的升级

项目注意事项
  1.按照规定时间完成项目
  2.项目中的人员和技术冲突

HTTPServer v3.0
  httpserver + WebFrame完成后端开发
  httpserver 功能:
    1.获取http请求
    2.解析http请求
    3.将请求内容发送给WebFrame
    4.从WebFrame接受返回数据
    5.将数据组织为Response格式发送给客户端

  WebFrame功能:
    1.从httpserver接收具体请求
    2.判断请求网页或者数据,根据请求调用函数处理
    3.将网页或者数据发送给httpserver

  升级点:
    1.采用了httpserver与应用程序分离,两个功能模块独立
      降低了耦合度
    2.数据的处理单独调用函数完成
    3.两者的配合使用进程间通信

电子词典
功能说明:
  1.用户可以登录和注册
    登录凭借用户名和密码登录
    注册要求用户必须填写用户名,密码,其它内容自己拟定
    用户名要求不能重复
  2.用户信息需要长期保存,以满足随时登录
    要求使用数据库保存
  3.能够满足多个用户同时登录操作的需要(并发操作)
  4.项目分为客户端和服务端,客户端负责请求和展示数据,其余处理由服务端完成
  5.客户端启动即进入一级界面(界面由print打印),
    一级界面功能:登录,注册,退出
  6.用户登陆后即进入二级界面(注册后是否处于登陆状态自定)
    二级界面功能:查单词 查看历史记录 注销
    查单词:可以循环输入单词,获取单词解释
      提示:每个单词占一行
          单词按照顺序排列
          单词和解释之间一定有空格
      可以选择直接操作文本,或者将单词本导入数据库
    查看历史记录:   
           格式:  name word time
           可以显示登录人员查的所有单词或者最近10条
    注销:返回一级界面

  电子词典库:
    用户信息表user:
      create table user(
      id int unsigned primary key auto_increment,
      name varchar(20),
      password  char(40),
      age tinyint unsigned,
      sex enum('m','f'),
      unique(name)
      );
    历史记录表history:
    create table history(
      id int unsigned primary key auto_increment,
      name varchar(20),
      word varchar(30),
      time timestamp
    );

    create table dictionary(
      id int unsigned primary key auto_increment,
      word varchar(30) ,
      statement varchar(1000),
      unique(word)
    );

3.结构设计,仿照ftp文件服务器
  使用类将功能进行封装
4.完成通信的搭建
  使用tcp套接字连接服务端
  服务端使用IO多路复用完成请求处理
5.功能分析,根据分析,完成功能代码

  网络通信:
    服务端:建立tcp套接字
          设置地址重用
          绑定地址
          设置监听套接字
          poll实现IO多路复用
          等待连接
    客户端:建立tcp套接字
          与服务器建立连接
          打印一级界面
          输入选项
  
  注册
    服务端:接收用户名
          判断用户名是否存在
          根据判断返回相应结果
          将用户密码经过hashi加密后存储
          将用户其它信息存入user表中
    客户端:打印一级界面,提示输入选项进行操作
          1登录 2注册 3退出
          选择注册,输入用户信息
          输入用户名后服务端接收信息判断是否存在并返回
          如果用户名可用则继续输入,否则重新输入
          注册完成后直接进入二级界面

  登录
    服务端:接收客户端请求
          搜索数据库是否有请求的用户名
          如果用户名不存在返回提示
          用户名存在将接收的密码进行加密后匹配
          返回匹配结果
          登录成功后进入查询状态

    客户端:打印一级界面,提示输入选项进行操作
          1登录 2注册 3退出
          选择登录
          输入用户名,即密码
          登录成功后清屏,并打印二级界面
          否则提示用户名不存在或密码有误

  查询单词:
    服务端:接收客户端请求
          查找字典表
          如果存在返回单词释义
          不存在返回'单词不存在'提示
          在表history中插入查询记录
    客户端:打印二级界面
          1.查单词 2.查看历史记录 3.注销
          输入选项1
          输入需要查询的单词,打包发送至服务端
          接收查询结果并显示
  
  查询历史记录:
    服务端:接收客户端请求
          查询history中对应用户名记录
          只返回10条记录
          
    客户端:打印二级界面
          1.查单词 2.查看历史记录 3.注销
          输入选项2
          发送查询记录请求
          接收并显示查询记录(最多10条)

  注销:
    服务端:接收客户端注销信息
          退出查询功能
          返回登录状态
    客户端:打印二级界面
          1.查单词 2.查看历史记录 3.注销
          输入选项3
          给服务端发送注销信息
          清屏
          返回一级界面


  退出:
    服务端:接收客户端请求
          断开连接,关闭套接字
    客户端:打印一级界面
          输入选项3退出
          向服务器发送退出请求
          关闭套接字

pip的使用
  作用:安装python的标准第三方库
  安装:sudo apt-get install python3-pip

  常用命令:
    安装包: pip3 install 库名

    查看python包:pip3 list

    搜索python包:pip3 search 库名

    升级软件包:pip3 install --upgrade 库名

    查看软件包详细信息: pip3 show 库名

    卸载软件包 pip3 uninstall 库名

    导出软件环境 pip3 freeze > 文件名(通常取为:requirements.txt)

    安装软件环境 pip3 install -r requirements.txt
